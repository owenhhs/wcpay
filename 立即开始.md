# 🎯 立即开始 - 选项B

## 当前情况

您当前在 **Mac** 环境中，需要进入 **OrbStack Ubuntu** 环境来运行测试脚本。

## 🚀 最简单的方法

我已经为您创建了一个一键启动脚本！

### 方法1: 使用一键启动脚本（推荐）

直接在Mac终端运行：

```bash
cd /Users/michael/Desktop/woocommerce-pay-20251122
bash quick-start-ubuntu.sh
```

这个脚本会：
- ✅ 自动检查OrbStack和Ubuntu实例
- ✅ 进入Ubuntu环境
- ✅ 找到项目目录
- ✅ 自动运行协助脚本

### 方法2: 手动步骤

如果您想手动操作：

**步骤1**: 打开新终端窗口，运行：
```bash
orbstack shell ubuntu
```

**步骤2**: 在Ubuntu环境中，进入项目目录：
```bash
# 尝试这些路径（选择存在的）
cd /mnt/Users/michael/Desktop/woocommerce-pay-20251122
# 或
cd /host/Users/michael/Desktop/woocommerce-pay-20251122
# 或使用git克隆
cd ~
git clone https://github.com/owenhhs/wcpay.git
cd wcpay
```

**步骤3**: 运行协助脚本：
```bash
chmod +x scripts/*.sh
bash scripts/assist-test.sh
```

## 📋 协助脚本菜单

运行协助脚本后，您会看到：

```
选择操作：
1. 检查环境状态
2. 设置开发环境（首次运行）  ← 首次运行选这个
3. 运行完整测试
4. 测试API连接
5. 查看日志
6. 手动测试步骤
7. 退出
```

## ✅ 建议的操作顺序

### 首次运行：

1. **选择 2** - 设置开发环境
   - 这会安装WordPress、Nginx、MySQL、PHP
   - 需要几分钟时间
   
2. **选择 3** - 运行完整测试
   - 检查所有组件
   - 创建测试产品
   
3. **选择 5** - 查看日志
   - 确保一切正常

### 后续运行：

直接选择 **3** 运行测试即可。

## 🔧 如果遇到问题

### 问题1: 找不到Ubuntu实例

```bash
# 创建Ubuntu实例
orbstack create ubuntu:22.04
```

### 问题2: 找不到项目目录

使用git克隆：
```bash
cd ~
git clone https://github.com/owenhhs/wcpay.git
cd wcpay
```

### 问题3: 脚本没有执行权限

```bash
chmod +x scripts/*.sh
```
